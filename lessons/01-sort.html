Bài 1: Thuật Toán Sắp Xếp

7 thuật toán sắp xếp từ đơn giản đến nâng cao. Mỗi thuật toán có: ý tưởng, giả mã, Python, mô phỏng, phân tích BigO.

📖 Bài toán sắp xếp là gì?

🔑 Định nghĩa

Cho một dãy gồm n phần tử a₁, a₂, ..., aₙ. Bài toán sắp xếp là sắp xếp lại các phần tử theo thứ tự tăng dần (hoặc giảm dần).

Mỗi phần tử aᵢ có một khóa (key) aᵢ.key dùng để so sánh. Khi sắp xếp, ta sắp các phần tử theo khóa của chúng.

⚠️ Độ phức tạp tối thiểu

Với bài toán sắp xếp dựa trên so sánh (comparison-based sorting), độ phức tạp tối thiểu là O(n log n). Nghĩa là KHÔNG có thuật toán sắp xếp nào tốt hơn O(n log n) nếu chỉ dùng phép so sánh giữa các phần tử.

Muốn nhanh hơn (O(n)) phải dùng thuật toán không so sánh như BinSort, Bucket Sort (dựa vào giá trị khóa).

1️⃣ Selection Sort (Sắp xếp chọn)

💡 Ý tưởng

Ở mỗi bước, tìm phần tử nhỏ nhất trong đoạn chưa sắp xếp và hoán vị nó với phần tử đầu tiên của đoạn chưa sắp xếp.

Sau bước i, i phần tử đầu tiên đã được sắp xếp đúng vị trí.

📋 Giả mã

PROCEDURE SelectionSort(VAR a: ARRAY[1..n] OF integer);
VAR i, j, lowindex: Integer;
    lowkey: Integer;
BEGIN
  FOR i := 1 TO n-1 DO
  BEGIN
    lowindex := i;
    lowkey := a[i];
    FOR j := i+1 TO n DO
      IF a[j] < lowkey THEN
      BEGIN
        lowkey := a[j];
        lowindex := j;
      END;
    Swap(a[i], a[lowindex]);
  END;
END;

🐍 Cài đặt Python

def selection_sort(a):
    n = len(a)
    for i in range(n - 1):
        lowindex = i
        for j in range(i + 1, n):
            if a[j] < a[lowindex]:
                lowindex = j
        a[i], a[lowindex] = a[lowindex], a[i]
    return a

# Test
arr = [5, 3, 8, 1, 9, 2, 7]
print(selection_sort(arr))
# Output: [1, 2, 3, 5, 7, 8, 9]

📊 Phân tích độ phức tạp

📐 Phân tích

Số lần so sánh: Vòng lặp ngoài chạy n-1 lần. Vòng lặp trong chạy n-i lần cho mỗi i. Tổng: (n-1) + (n-2) + ... + 1 = n(n-1)/2.

Số lần hoán vị: Tối đa n-1 lần (mỗi bước 1 lần).

Độ phức tạp: T(n) = n(n-1)/2 = O(n²) trong mọi trường hợp (tốt/xấu/trung bình).

Bộ nhớ: O(1) — sắp xếp tại chỗ.

🎯 Ví dụ chạy tay

📝 Mảng: [5, 3, 8, 1, 9]

Bước 1: Tìm min trong [5,3,8,1,9] → 1 ở vị trí 3. Swap với vị trí 0.

[5, 3, 8, 1, 9] → [1, 3, 8, 5, 9]   (swap a[0]↔a[3])

Bước 2: Tìm min trong [3,8,5,9] → 3 ở vị trí 1. Đã đúng vị trí → không swap.

[1, 3, 8, 5, 9]   (không đổi)

Bước 3: Tìm min trong [8,5,9] → 5 ở vị trí 3. Swap với vị trí 2.

[1, 3, 8, 5, 9] → [1, 3, 5, 8, 9]

Bước 4: Tìm min trong [8,9] → 8. Đã đúng → không swap.

[1, 3, 5, 8, 9]   ✓ HOÀN THÀNH

2️⃣ Insertion Sort (Sắp xếp chèn)

💡 Ý tưởng

Ở mỗi bước, lấy phần tử tiếp theo và chèn nó vào vị trí thích hợp trong đoạn đã sắp xếp.

Giống như sắp bài trên tay — cứ rút 1 lá rồi chèn vào đúng vị trí trong số các lá đã có.

📋 Giả mã

PROCEDURE InsertionSort(VAR a: ARRAY[1..n] OF integer);
VAR i, j: Integer;
BEGIN
  FOR i := 1 TO n-1 DO
  BEGIN
    j := i;
    WHILE (j >= 1) AND (a[j] < a[j-1]) DO
    BEGIN
      Swap(a[j], a[j-1]);
      j := j - 1;
    END;
  END;
END;

🐍 Cài đặt Python

def insertion_sort(a):
    n = len(a)
    for i in range(1, n):
        j = i
        while j >= 1 and a[j] < a[j - 1]:
            a[j], a[j - 1] = a[j - 1], a[j]
            j -= 1
    return a

# Test
arr = [5, 3, 8, 1, 9]
print(insertion_sort(arr))
# Output: [1, 3, 5, 8, 9]

📊 Phân tích độ phức tạp

📐 Phân tích 3 trường hợp

Trường hợp TỐT NHẤT (mảng đã sắp): Vòng while không bao giờ chạy (a[j] ≥ a[j-1]). Chỉ còn vòng for → O(n).

Trường hợp XẤU NHẤT (mảng sắp ngược): Vòng while chạy tối đa i lần cho mỗi i. Tổng: 1+2+...+(n-1) = n(n-1)/2 → O(n²).

Trung bình: Vòng while chạy i/2 lần trung bình → vẫn O(n²).

Bộ nhớ: O(1).

💡 Khi nào dùng Insertion Sort?

Insertion Sort rất hiệu quả khi:

  • Mảng đã gần sắp xếp (chỉ vài phần tử lệch) — gần O(n).
  • Mảng nhỏ (n < 50) — overhead thấp hơn QuickSort.
  • Dùng làm bước cơ sở trong QuickSort (khi đoạn nhỏ đủ nhỏ, dùng Insertion Sort).

🎯 Ví dụ chạy tay

📝 Mảng: [5, 3, 8, 1]

Bước 1: Lấy a[1]=3. So sánh với a[0]=5 → 3<5 → swap.

[5, 3, 8, 1] → [3, 5, 8, 1]

Bước 2: Lấy a[2]=8. So sánh với a[1]=5 → 8>5 → dừng. Không swap.

[3, 5, 8, 1]   (không đổi)

Bước 3: Lấy a[3]=1. So sánh với a[2]=8 → 1<8 → swap. Tiếp tục với a[1]=5 → swap. Tiếp tục với a[0]=3 → swap.

[3, 5, 8, 1] → [3, 5, 1, 8] → [3, 1, 5, 8] → [1, 3, 5, 8]   ✓

3️⃣ Bubble Sort (Sắp xếp nổi bọt)

💡 Ý tưởng

Lặp lại việc duyệt qua dãy, hoán vị các cặp phần tử liền kề nếu chúng sai thứ tự.

Sau mỗi lần duyệt, phần tử lớn nhất "nổi" lên cuối dãy (như bong bóng nổi lên).

📋 Giả mã

PROCEDURE BubbleSort(VAR a: ARRAY[1..n] OF integer);
VAR i, j: Integer;
BEGIN
  FOR i := 1 TO n-1 DO
    FOR j := n DOWNTO i+1 DO
      IF a[j-1] > a[j] THEN
        Swap(a[j-1], a[j]);
END;

🐍 Cài đặt Python (có tối ưu flag)

def bubble_sort(a):
    n = len(a)
    for i in range(n - 1):
        swapped = False
        for j in range(n - 1, i, -1):
            if a[j - 1] > a[j]:
                a[j - 1], a[j] = a[j], a[j - 1]
                swapped = True
        if not swapped:  # Tối ưu: nếu không có swap → đã sắp xong
            break
    return a

# Test
arr = [5, 3, 8, 1, 9]
print(bubble_sort(arr))  # [1, 3, 5, 8, 9]

📊 Phân tích

📐 Phân tích

TỐT NHẤT (đã sắp, có tối ưu flag): Vòng for trong chạy 1 lần không có swap → thoát sớm → O(n).

XẤU NHẤT (sắp ngược): Tổng số lần so sánh = (n-1)+(n-2)+...+1 = n(n-1)/2 → O(n²).

Trung bình: O(n²).

Bộ nhớ: O(1).

4️⃣ QuickSort — Thuật toán quan trọng nhất

💡 Ý tưởng (Chia để trị)

1. Chọn pivot (trục) — thường là phần tử giữa.

2. Phân hoạch (Partition): Sắp xếp sao cho mọi phần tử bên trái pivot ≤ pivot, mọi phần tử bên phải ≥ pivot.

3. Đệ quy sắp xếp 2 phần bên trái và bên phải pivot.

📋 Giả mã QuickSort

PROCEDURE QuickSort(VAR a: ARRAY[1..n] OF integer; l, r: Integer);
VAR i, j, pivot: Integer;
BEGIN
  IF l < r THEN
  BEGIN
    i := l; j := r;
    pivot := a[(l + r) DIV 2];   { Chọn pivot ở giữa }
    REPEAT
      WHILE a[i] < pivot DO i := i + 1;
      WHILE a[j] > pivot DO j := j - 1;
      IF i <= j THEN
      BEGIN
        Swap(a[i], a[j]);
        i := i + 1; j := j - 1;
      END;
    UNTIL i > j;
    IF l < j THEN QuickSort(a, l, j);   { Đệ quy trái }
    IF i < r THEN QuickSort(a, i, r);   { Đệ quy phải }
  END;
END;

🐍 Cài đặt Python

def quick_sort(a, l, r):
    if l < r:
        i, j = l, r
        pivot = a[(l + r) // 2]
        while i <= j:
            while a[i] < pivot:
                i += 1
            while a[j] > pivot:
                j -= 1
            if i <= j:
                a[i], a[j] = a[j], a[i]
                i += 1
                j -= 1
        if l < j:
            quick_sort(a, l, j)
        if i < r:
            quick_sort(a, i, r)
    return a

# Test
arr = [5, 3, 8, 1, 9, 2, 7]
print(quick_sort(arr, 0, len(arr) - 1))
# Output: [1, 2, 3, 5, 7, 8, 9]

📊 Phân tích QuickSort

📐 Phân tích 3 trường hợp

TỐT NHẤT: Mỗi lần phân hoạch chia mảng làm 2 nửa gần bằng nhau.

Phương trình đệ quy: T(n) = 2T(n/2) + O(n) → O(n log n).

XẤU NHẤT: Pivot luôn là phần tử lớn nhất/nhỏ nhất → phân hoạch lệch hoàn toàn (1 bên rỗng, 1 bên n-1).

Phương trình: T(n) = T(n-1) + O(n) → O(n²).

Xảy ra khi mảng đã sắp xếp và chọn pivot sai (đầu/cuối).

TRUNG BÌNH: O(n log n) — đây là lý do QuickSort được dùng rộng rãi.

Bộ nhớ: O(log n) cho stack đệ quy.

⚠️ Bài toán con cân bằng

Sẽ tốt hơn nếu ta chia bài toán thành các bài toán con có kích thước gần bằng nhau.

Ví dụ: MergeSort chia mảng thành 2 nửa bằng nhau → luôn O(n log n).

QuickSort trong trường hợp xấu chia lệch hoàn toàn → O(n²).

Nguyên tắc chung: Chia bài toán thành các bài toán con có kích thước xấp xỉ bằng nhau thì hiệu suất sẽ cao hơn.

🎯 Ví dụ chạy tay QuickSort

📝 Mảng: [5, 3, 8, 1, 9, 2, 7]

Lần 1: l=0, r=6. Pivot = a[3] = 1.

Phân hoạch: i=0 (5≥1 dừng), j=6→...→3 (a[3]=1 không > 1 dừng). Swap a[0]↔a[3]: [1, 3, 8, 5, 9, 2, 7]. i=1, j=2.

i=1 (a[1]=3≥1 dừng), j=2 (a[2]=8>1 j--=1). i>j → dừng.

→ Đệ quy trái [0..2] và phải [1..6].

(Tiếp tục... mỗi bước con chạy như trên.)

Kết quả cuối: [1, 2, 3, 5, 7, 8, 9] ✓

5️⃣ HeapSort — Dùng cấu trúc Heap

🔑 Cấu trúc Heap

Heap là cây nhị phân đầy đủ có tính chất: giá trị tại mỗi nút lớn hơn hoặc bằng (hoặc nhỏ hơn hoặc bằng) giá trị tại các nút con.

Max-heap: Nút cha ≥ nút con. Gốc chứa phần tử lớn nhất.

Min-heap: Nút cha ≤ nút con. Gốc chứa phần tử nhỏ nhất.

💡 Ý tưởng HeapSort (2 giai đoạn)

Giai đoạn 1 — Xây dựng heap: Biến mảng thành max-heap (phần tử lớn nhất ở gốc a[1]).

Giai đoạn 2 — Rút gốc: Lần lượt lấy phần tử lớn nhất (gốc heap) ra, đặt vào cuối mảng, rồi phục hồi tính chất heap cho phần còn lại.

📋 Giả mã HeapSort + Heapify

PROCEDURE Heapify(VAR a: ARRAY[1..n] OF integer; i, n: Integer);
VAR largest, left, right: Integer;
BEGIN
  largest := i;
  left := 2 * i;
  right := 2 * i + 1;
  IF (left <= n) AND (a[left] > a[largest]) THEN
    largest := left;
  IF (right <= n) AND (a[right] > a[largest]) THEN
    largest := right;
  IF largest <> i THEN
  BEGIN
    Swap(a[i], a[largest]);
    Heapify(a, largest, n);   { Đệ quy phục hồi }
  END;
END;

PROCEDURE HeapSort(VAR a: ARRAY[1..n] OF integer);
VAR i: Integer;
BEGIN
  { Giai đoạn 1: Xây heap }
  FOR i := n DIV 2 DOWNTO 1 DO
    Heapify(a, i, n);
  { Giai đoạn 2: Rút gốc }
  FOR i := n DOWNTO 2 DO
  BEGIN
    Swap(a[1], a[i]);       { Đưa gốc (max) về cuối }
    Heapify(a, 1, i - 1);   { Phục hồi heap cho phần còn lại }
  END;
END;

🐍 Cài đặt Python (0-indexed)

def heapify(a, n, i):
    """Phục hồi tính chất max-heap cho cây con gốc i."""
    largest = i
    left = 2 * i + 1   # 0-indexed: left = 2i+1
    right = 2 * i + 2  # right = 2i+2
    if left < n and a[left] > a[largest]:
        largest = left
    if right < n and a[right] > a[largest]:
        largest = right
    if largest != i:
        a[i], a[largest] = a[largest], a[i]
        heapify(a, n, largest)

def heap_sort(a):
    n = len(a)
    # Giai đoạn 1: Xây heap (từ n//2 - 1 xuống 0)
    for i in range(n // 2 - 1, -1, -1):
        heapify(a, n, i)
    # Giai đoạn 2: Rút gốc
    for i in range(n - 1, 0, -1):
        a[0], a[i] = a[i], a[0]  # Đưa max về cuối
        heapify(a, i, 0)         # Phục hồi heap cho phần [0..i-1]
    return a

# Test
arr = [5, 3, 8, 1, 9]
print(heap_sort(arr))  # [1, 3, 5, 8, 9]

📊 Phân tích HeapSort

📐 Phân tích

Xây dựng heap: O(n) — gọi heapify từ n/2 xuống 1, tổng thời gian tuyến tính.

Mỗi lần rút gốc + heapify: O(log n) — heapify đi từ gốc xuống lá (chiều cao log n).

Tổng: O(n) + n × O(log n) = O(n log n) trong mọi trường hợp (tốt/xấu/trung bình).

Bộ nhớ: O(1) — sắp xếp tại chỗ.

💡 Ưu điểm HeapSort

• Đảm bảo O(n log n) trong mọi trường hợp (không như QuickSort có thể O(n²)).

• Sắp xếp tại chỗ (O(1) bộ nhớ).

• Tuy nhiên thực tế chậm hơn QuickSort do cache-unfriendly (truy cập heap nhảy xa).

6️⃣ BinSort (Sắp xếp theo xô)

💡 Ý tưởng

Nếu các phần tử có khóa trong phạm vi nhỏ (từ 1 đến k), ta có thể sắp xếp trong thời gian O(n+k).

Tạo mảng đếm b[1..k], đếm số lần xuất hiện của từng giá trị, rồi viết lại mảng theo thứ tự.

📋 Giả mã

PROCEDURE BinSort(VAR a: ARRAY[1..n] OF integer; k: Integer);
VAR b: ARRAY[1..k] OF integer;
    i, j: Integer;
BEGIN
  FOR i := 1 TO k DO b[i] := 0;        { Khởi tạo xô rỗng }
  FOR i := 1 TO n DO b[a[i]] := b[a[i]] + 1;  { Đếm }
  j := 1;
  FOR i := 1 TO k DO                    { Viết lại mảng }
    WHILE b[i] > 0 DO
    BEGIN
      a[j] := i;
      j := j + 1;
      b[i] := b[i] - 1;
    END;
END;

🐍 Cài đặt Python

def bin_sort(a, k):
    """k = giá trị lớn nhất có thể có trong a."""
    b = [0] * (k + 1)  # b[0..k], chỉ dùng b[1..k]
    for x in a:
        b[x] += 1
    j = 0
    for i in range(1, k + 1):
        while b[i] > 0:
            a[j] = i
            j += 1
            b[i] -= 1
    return a

# Test: mảng các giá trị 1-9
arr = [5, 3, 8, 1, 9, 3, 7]
print(bin_sort(arr, 9))  # [1, 3, 3, 5, 7, 8, 9]

📊 Phân tích

📐 Phân tích

T(n) = O(n + k) — phải duyệt qua mảng n phần tử + duyệt qua k xô.

Nếu k = O(n) (giá trị khóa cùng bậc với n) thì T(n) = O(n) — nhanh hơn mọi thuật toán so sánh!

Bộ nhớ: O(k) — mảng đếm.

Điều kiện: Chỉ dùng được khi khóa là số nguyên trong phạm vi nhỏ.

7️⃣ Bucket Sort

💡 Ý tưởng

Khi các phần tử là các số thực trong khoảng [0, 1), ta dùng Bucket Sort:

  1. Chia khoảng [0, 1) thành n xô (bucket) bằng nhau (mỗi xô rộng 1/n).
  2. Đưa mỗi phần tử vào xô tương ứng (xô ⌊n·a[i]⌋).
  3. Sắp xếp từng xô (dùng Insertion Sort — xô thường ít phần tử).
  4. Nối các xô lại theo thứ tự.

🐍 Cài đặt Python

def bucket_sort(a):
    n = len(a)
    buckets = [[] for _ in range(n)]
    # 1. Phân phối vào xô
    for x in a:
        idx = min(int(x * n), n - 1)  # Đảm bảo không vượt biên
        buckets[idx].append(x)
    # 2. Sắp xếp từng xô (Insertion Sort)
    for b in buckets:
        b.sort()  # Python dùng Timsort, gần ý tưởng Insertion
    # 3. Nối các xô
    result = []
    for b in buckets:
        result.extend(b)
    return result

# Test: số thực trong [0, 1)
arr = [0.78, 0.17, 0.39, 0.26, 0.72, 0.94, 0.21, 0.12, 0.23, 0.68]
print(bucket_sort(arr))
# [0.12, 0.17, 0.21, 0.23, 0.26, 0.39, 0.68, 0.72, 0.78, 0.94]

📊 Phân tích

📐 Phân tích

Trung bình: O(n) — giả sử phần tử phân bố đều, mỗi xô có trung bình 1 phần tử.

XẤU NHẤT: O(n²) — mọi phần tử rơi vào cùng 1 xô → Insertion Sort trên n phần tử.

Bộ nhớ: O(n) — cần n xô.

✍️ Bài tập tự giải (có đáp án chi tiết)

📝 Bài 1

Cho mảng a = [4, 2, 7, 1, 3]. Hãy chạy tay Selection Sort qua từng bước.

📌 Công thức áp dụng: Selection Sort — mỗi bước tìm min trong đoạn chưa sắp, swap với đầu đoạn.

🧮 Tính toán từng bước:

Bước 1 (i=0): Tìm min trong [4,2,7,1,3] → 1 ở vị trí 3. Swap a[0]↔a[3]: [4,2,7,1,3] → [1,2,7,4,3] Bước 2 (i=1): Tìm min trong [2,7,4,3] → 2 ở vị trí 1. Đã đúng. Không swap: [1,2,7,4,3] Bước 3 (i=2): Tìm min trong [7,4,3] → 3 ở vị trí 4. Swap a[2]↔a[4]: [1,2,7,4,3] → [1,2,3,4,7] Bước 4 (i=3): Tìm min trong [4,7] → 4 ở vị trí 3. Đã đúng. Không swap: [1,2,3,4,7]

✓ Kết quả: [1, 2, 3, 4, 7] sau 4 bước, 2 lần swap.

🔍 Kiểm tra: Mảng đã tăng dần ✓. Độ phức tạp: n(n-1)/2 = 5·4/2 = 10 lần so sánh = O(n²).

📝 Bài 2

Cho mảng a = [3, 1, 4, 1, 5, 9, 2, 6]. Chạy QuickSort 1 bước phân hoạch đầu tiên (l=0, r=7, pivot = a[3] = 1). Cho biết mảng sau phân hoạch và 2 đoạn cần đệ quy.

📌 Công thức áp dụng: QuickSort phân hoạch — 2 con trỏ i, j di chuyển từ 2 đầu vào giữa, swap khi gặp cặp sai vị trí.

🧮 Tính toán từng bước:

l=0, r=7. Pivot = a[(0+7)/2] = a[3] = 1. i=0, j=7. Bước lặp: - a[i]=a[0]=3, 3 < 1? Không. i dừng tại 0. - a[j]=a[7]=6, 6 > 1? Có. j-- → j=6. - a[j]=a[6]=2, 2 > 1? Có. j-- → j=5. - a[j]=a[5]=9, 9 > 1? Có. j-- → j=4. - a[j]=a[4]=5, 5 > 1? Có. j-- → j=3. - a[j]=a[3]=1, 1 > 1? Không. j dừng tại 3. - i=0 <= j=3 → swap a[0]↔a[3]: [3,1,4,1,5,9,2,6] → [1,1,4,3,5,9,2,6] i=1, j=2. - a[i]=a[1]=1, 1 < 1? Không. i dừng tại 1. - a[j]=a[2]=4, 4 > 1? Có. j-- → j=1. - a[j]=a[1]=1, 1 > 1? Không. j dừng tại 1. - i=1 <= j=1 → swap a[1]↔a[1]: (không đổi). i=2, j=0. - i=2 > j=0 → dừng. Mảng sau phân hoạch: [1, 1, 4, 3, 5, 9, 2, 6] Đoạn trái cần đệ quy: l=0..j=0 → [1] (1 phần tử, không cần) Đoạn phải cần đệ quy: i=2..r=7 → [4, 3, 5, 9, 2, 6]

✓ Kết quả: Mảng [1, 1, 4, 3, 5, 9, 2, 6]. Đoạn phải [4, 3, 5, 9, 2, 6] cần đệ quy tiếp.

🔍 Kiểm tra: Phần tử a[0]=a[1]=1 ≤ pivot, a[2..7] ≥ pivot. Phân hoạch đúng.

📝 Bài 3

Cho mảng a = [4, 10, 3, 5, 1] (1-indexed). Xây max-heap từ mảng này. Cho biết mảng sau khi xây heap.

📌 Công thức áp dụng: Heapify từ n/2 xuống 1. Tại mỗi nút i, so sánh a[i] với a[2i] (left) và a[2i+1] (right), swap với con lớn hơn nếu cần, đệ quy xuống.

🧮 Tính toán từng bước (1-indexed):

Mảng ban đầu: [4, 10, 3, 5, 1] (vị trí 1, 2, 3, 4, 5) n=5, bắt đầu heapify từ i = n/2 = 2 xuống 1. Heapify(i=2): nút a[2]=10, left=a[4]=5, right=a[5]=1. largest=2 (a[2]=10 ≥ a[4]=5, ≥ a[5]=1) → không swap. Heapify(i=1): nút a[1]=4, left=a[2]=10, right=a[3]=3. largest=2 (a[2]=10 > a[1]=4). Swap a[1]↔a[2]: Mảng: [10, 4, 3, 5, 1] Đệ quy Heapify(i=2): nút a[2]=4, left=a[4]=5, right=a[5]=1. largest=4 (a[4]=5 > a[2]=4). Swap a[2]↔a[4]: Mảng: [10, 5, 3, 4, 1] Đệ quy Heapify(i=4): a[4]=4 không có con (2*4=8 > n=5) → dừng. Kết quả: max-heap = [10, 5, 3, 4, 1] 10 / \ 5 3 / \ 4 1

✓ Kết quả: Mảng sau khi xây heap: [10, 5, 3, 4, 1].

🔍 Kiểm tra: Mỗi nút cha ≥ nút con: 10≥5,3; 5≥4,1. ✓ Đúng max-heap.

📝 Bài 4

Phân tích độ phức tạp QuickSort trong trường hợp tốt nhất, xấu nhất, trung bình. Viết phương trình đệ quy.

📌 Công thức áp dụng: Phân tích đệ quy — T(n) = a·T(n/b) + f(n). Số phần tử giảm theo b, mỗi lần chia thành a bài toán con, mất f(n) để phân hoạch.

🧮 Tính toán từng trường hợp:

1. TỐT NHẤT (chia đều 2 nửa): T(n) = 2T(n/2) + O(n) Áp dụng Định lý Master: a=2, b=2, f(n)=n → n^(log_b a) = n^1 = n f(n) = Θ(n^1) → case 2 → T(n) = O(n log n) 2. XẤU NHẤT (chia lệch 1-1): Pivot luôn là min hoặc max → 1 bên rỗng, 1 bên n-1. T(n) = T(n-1) + O(n) Giải: T(n) = T(n-1) + n = T(n-2) + (n-1) + n = ... = 1 + 2 + ... + n = n(n+1)/2 → T(n) = O(n²) 3. TRUNG BÌNH: Pivot chia theo tỉ lệ α:(1-α) với α ngẫu nhiên. T(n) = T(αn) + T((1-α)n) + O(n) → T(n) = O(n log n) với α bất kỳ ∈ (0,1) cố định.

✓ Kết quả: Tốt O(n log n), Xấu O(n²), Trung bình O(n log n).

🔍 Kiểm tra: QuickSort thực tế nhanh hơn HeapSort (cùng O(n log n)) vì cache-friendly — truy cập mảng tuần tự, không nhảy xa như heap.

❓ Kiểm tra hiểu bài

🧠 Tư duy thuật toán sắp xếp

🏆 Cách tư duy để giải bài lạ

1. Khi gặp bài "sắp xếp + điều kiện gì đó"

• Nếu chỉ cần sắp xếp tăng/giảm → dùng thư viện (Python sort() dùng Timsort — O(n log n)).

• Nếu yêu cầu "không dùng thư viện" → QuickSort hoặc HeapSort (đều O(n log n)).

• Nếu mảng gần sắp xếp → Insertion Sort (gần O(n)).

• Nếu mảng nhỏ (n < 50) → Insertion Sort (overhead thấp).

• Nếu khóa là số nguyên nhỏ → BinSort (O(n+k)).

• Nếu là số thực [0,1) → Bucket Sort.

2. Khi cần chọn thuật toán cho đề thi

• Đề yêu cầu O(n log n) ổn định → HeapSort (không sợ xấu nhất).

• Đề yêu cầu sắp xếp tại chỗ (O(1) bộ nhớ) → HeapSort hoặc QuickSort.

• Đề yêu cầu ổn định (stable sort) → Insertion Sort, MergeSort (KHÔNG phải Selection/QuickSort).

• Đề cho dữ liệu đặc biệt (số nguyên nhỏ, real [0,1)) → BinSort/Bucket.

3. Khi gặp bài "tìm k phần tử lớn nhất/nhỏ nhất"

• Nếu k nhỏ → dùng Heap (min-heap k phần tử, O(n log k)).

• Nếu k ≈ n/2 → sắp xếp xong lấy O(n log n).

• QuickSelect — biến thể QuickSort — trung bình O(n).

4. Bài "kiểm tra 2 mảng có là hoán vị không"

• Sắp xếp cả 2 → so sánh → O(n log n).

• Hoặc dùng hash map đếm tần suất → O(n).

🏆 Cách trình bày bài thi

📋 Mẫu trình bày chuẩn cho bài sắp xếp

Đề bài mẫu: "Trình bày giải thuật QuickSort. Phân tích độ phức tạp."

Trình bày chuẩn:

1. Ý TƯỞNG:
   QuickSort dựa trên kỹ thuật chia để trị:
   - Chọn 1 phần tử làm pivot (thường là phần tử giữa).
   - Phân hoạch mảng thành 2 phần: bên trái ≤ pivot, bên phải ≥ pivot.
   - Đệ quy sắp xếp 2 phần.

2. GIẢ MÃ:
   PROCEDURE QuickSort(VAR a; l, r);
   BEGIN
     IF l < r THEN
       i := l; j := r;
       pivot := a[(l+r) DIV 2];
       REPEAT
         WHILE a[i] < pivot DO i++;
         WHILE a[j] > pivot DO j--;
         IF i <= j THEN BEGIN Swap(a[i], a[j]); i++; j--; END;
       UNTIL i > j;
       QuickSort(a, l, j);
       QuickSort(a, i, r);
     END;
   END;

3. VÍ DỤ MINH HỌA:
   Mảng [5, 3, 8, 1, 9]. Pivot = a[2] = 8.
   (chạy tay từng bước...)

4. PHÂN TÍCH ĐỘ PHỨC TẠP:
   - Tốt nhất: T(n) = 2T(n/2) + O(n) → O(n log n).
   - Xấu nhất: T(n) = T(n-1) + O(n) → O(n²) (mảng đã sắp, pivot sai).
   - Trung bình: O(n log n).
   - Bộ nhớ: O(log n) (stack đệ quy).

5. NHẬN XÉT:
   - Thực tế nhanh hơn HeapSort (cache-friendly).
   - Không ổn định (có thể đổi vị trí tương đối các phần tử bằng nhau).
   - Dùng trong hàm sort() của nhiều ngôn ngữ.
💡 5 bí kíp để được điểm tối đa
  1. Luôn viết "1. Ý TƯỞNG" trước — giám khảo xem ý tưởng trước khi đọc mã.
  2. Giả mã dùng cú pháp rõ ràng — PROCEDURE/BEGIN/END hoặc def/for/if Python.
  3. Cho 1 ví dụ nhỏ chạy tay — chứng tỏ hiểu thuật toán không chỉ copy mã.
  4. Phân tích 3 trường hợp: tốt/xấu/trung bình — viết phương trình đệ quy để chứng minh.
  5. Cuối bài nhận xét ưu/nhược điểm — được điểm cộng nếu so sánh với thuật toán khác.
⚠️ Lỗi mất điểm thường gặp
  • ❌ Chỉ viết mã không có ý tưởng → mất 30-40% điểm.
  • ❌ Không phân tích độ phức tạp → mất 20% điểm.
  • ❌ Quên trường hợp xấu nhất (QuickSort O(n²)) → mất điểm phân tích.
  • ❌ Nhầm Bubble Sort với Insertion Sort (cả 2 đều swap cặp liền kề, nhưng Bubble đi từ cuối lên, Insertion từ vị trí hiện tại lùi về).
  • ❌ HeapSort 0-indexed vs 1-indexed: chú ý index khi cài đặt (left = 2i+1 nếu 0-indexed, 2i nếu 1-indexed).

📌 Tóm tắt cuối bài

✅ Điểm cần nhớ
  1. Sắp xếp so sánh tối thiểu O(n log n). Không so sánh có thể O(n).
  2. Selection Sort: O(n²) mọi TH — chọn min hoán vị đầu.
  3. Insertion Sort: O(n) tốt, O(n²) xấu — chèn vào đoạn đã sắp.
  4. Bubble Sort: O(n²) — nổi bọt lớn lên cuối.
  5. QuickSort: O(n log n) trung bình, O(n²) xấu — chia để trị với pivot.
  6. HeapSort: O(n log n) mọi TH — dùng max-heap, rút gốc.
  7. BinSort: O(n+k) — đếm tần suất, cho số nguyên nhỏ.
  8. Bucket Sort: O(n) trung bình — chia xô cho số thực [0,1).
🎯 Mẹo thi nhanh

• QuickSort và HeapSort là 2 thuật toán hay ra nhất — phải biết viết + phân tích.

• Insertion Sort có ưu điểm O(n) khi gần sắp — hay dùng làm base case.

• BinSort/Bucket Sort là thuật toán "đặc biệt" — chỉ dùng được khi dữ liệu thỏa mãn điều kiện.

• Khi nghi ngờ → dùng QuickSort (tốt nhất thực tế) hoặc HeapSort (đảm bảo không xấu).
