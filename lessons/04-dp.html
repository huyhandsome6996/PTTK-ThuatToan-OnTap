Bài 4: Quy Hoạch Động (Dynamic Programming)

Quy hoạch động (DP) là kỹ thuật mạnh nhất trong PTTK — giải bài toán tối ưu bằng cách GHI NHỚ lời giải các bài toán con. 5 bài toán kinh điển: C(n,k), LIS, LCS, 0/1 Knapsack, phân tích số.

📖 Lý thuyết Quy hoạch động

🔑 Nguyên lý

DP giống chia để trị — chia bài toán thành các bài toán con nhỏ hơn. KHÁC BIỆT cốt lõi: các bài toán con KHÔNG độc lập với nhau (cùng dùng chung bài toán con nhỏ hơn), nên nếu dùng chia để trị sẽ lặp lại việc giải → DP giải 1 lần rồi GHI NHỚ.

Ví dụ kinh điển: Fibonacci. $F(n) = F(n-1) + F(n-2)$. Cả $F(n-1)$ và $F(n-2)$ đều cần $F(n-2)$, $F(n-3)$... Nếu đệ quy thông thường, $F(2)$ được tính đi tính lại rất nhiều lần. DP lưu $F(2)$ vào bảng → chỉ tính 1 lần.

📋 Phương pháp chung 3 bước

  1. Phân rãChia bài toán thành các bài toán con nhỏ hơn. Mỗi bài toán lớn được biểu diễn qua bài toán con.
  2. Ghi nhớ lời giảiLưu lời giải các bài toán con vào bảng (1D, 2D, 3D... tùy bài). Khi cần lại, tra cứu bảng thay vì tính lại.
  3. Tổng hợp lời giảiĐi từ dưới lên (bottom-up): từ bài toán con nhỏ nhất → lớn dần → bài toán gốc.

🛠️ Các bước thực hiện DP

📐 3 bước cài đặt

Bước 1: Lập hệ thức — tìm công thức truy hồi $F[i] = ...$ dựa vào $F[j]$ với $j < i$.

Bước 2: Tổ chức dữ liệu và chương trình — chọn bảng (1D/2D/3D), khởi tạo giá trị biên, tính bảng theo công thức từ dưới lên.

Bước 3: Làm mịn (nếu cần) — tối ưu bộ nhớ (vd giảm 2D → 1D), tối ưu thời gian.

🧠 Tư duy DP — Khi nào dùng?

💡 Dùng DP khi bài toán có 2 tính chất

1. Cấu trúc con tối ưu (subproblem optimality): Lời giải tối ưu của bài toán lớn chứa lời giải tối ưu của các bài toán con. Ví dụ: LIS của cả dãy chứa LIS của tiền tố.

2. Bài toán con chồng lặp (overlapping subproblems): Cùng 1 bài toán con được dùng nhiều lần trong quá trình giải. Đây là điều khiến chia để trị kém hiệu quả và DP tỏa sáng.

🔍 Cách nhận diện bài toán DP

📝 Dấu hiệu nhận biết
  • Câu hỏi dạng "Đếm số cách..." → thường DP. (vd: đếm số cách leo cầu thang, đếm số cách phân tích số).
  • Câu hỏi dạng "Tìm max/min..." với ràng buộc → thường DP. (vd: LIS, LCS, Knapsack).
  • Bài có công thức truy hồi rõ ràng → DP. (vd: Fibonacci, C(n,k)).
  • Bài mà đệ quy thông thường chậm do lặp → DP.

📝 5 bước giải bài DP (chuẩn trình bày thi)

  1. Định nghĩa F[i] (hoặc F[i,j]) là gìVí dụ: $F[i]$ = độ dài LIS kết thúc tại $a[i]$. Đây là bước quan trọng nhất — định nghĩa sai thì cả bài sai.
  2. Tìm công thức truy hồiNhìn bài toán con nhỏ nhất: $F[i]$ được tính từ các $F[j]$ nào? Điều kiện gì?
  3. Xác định giá trị biên$F[0]$, $F[1]$, $F[i,0]$... — giá trị khởi tạo để chạy công thức truy hồi.
  4. Tính từ dưới lên (bottom-up)Viết vòng lặp for đi từ $i=0$ đến $i=n$, dùng công thức truy hồi.
  5. Truy vết nếu cầnNếu đề yêu cầu chỉ ra các phần tử được chọn (không chỉ độ dài/giá trị), phải truy vết ngược từ $F[n]$ về.

1️⃣ Bài toán 1: Tính số tổ hợp C(n, k)

💡 Bài toán

Tính số tổ hợp chập $k$ của $n$: $C(n, k) = \frac{n!}{k!(n-k)!}$. Dùng công thức giai thừa trực tiếp sẽ tràn số và chậm. DP giúp tính nhanh mà không cần giai thừa.

📐 Công thức truy hồi

$$C(n, 0) = C(n, n) = 1 \quad \text{(biên)}$$

$$C(n, k) = C(n-1, k-1) + C(n-1, k) \quad \text{với } 0 < k < n$$

Giải thích: Xét 1 phần tử bất kỳ trong $n$ phần tử. Có 2 trường hợp chọn $k$ phần tử:

  • Không chứa phần tử đó → chọn $k$ phần tử từ $n-1$ phần tử còn lại: $C(n-1, k)$ cách.
  • Có chứa phần tử đó → chọn thêm $k-1$ phần tử từ $n-1$ phần tử còn lại: $C(n-1, k-1)$ cách.

Cộng lại: $C(n, k) = C(n-1, k-1) + C(n-1, k)$. ✓

🐢 Đệ quy thông thường — O(2ⁿ)

int C(int n, int k) {
    if (k == 0 || k == n) return 1;
    return C(n-1, k-1) + C(n-1, k);
}

Nhận xét: Đệ quy tính đi tính lại nhiều $C(i, j)$ — đặc biệt $C(0, 0)$ được tính $2^n$ lần. Độ phức tạp $O(2^n)$ — rất chậm. Đây là lúc DP phát huy tác dụng.

🚀 DP — Lập bảng 2D

Lập bảng $C[n+1][n+1]$ với $C[i][j] = C(i, j)$. Điền từ trên xuống dưới, trái qua phải.

Giả mã DP (C)

int Comb(int n, int k) {
    int C[n+1][n+1], i, j;
    C[0][0] = 1;
    for (i = 1; i <= n; i++) {
        C[i][0] = 1;          // biên trái
        C[i][i] = 1;          // biên chéo
        for (j = 1; j < i; j++)
            C[i][j] = C[i-1][j-1] + C[i-1][j];   // công thức truy hồi
    }
    return C[n][k];
}

📊 Phân tích độ phức tạp

Số phép tính: $\sum_{i=1}^{n}(i-1) = 0 + 1 + 2 + ... + (n-1) = \frac{n(n-1)}{2}$.

Độ phức tạp thời gian: $T(n) = \frac{n(n-1)}{2} = $ O(n²).

Độ phức tạp bộ nhớ: O(n²) cho bảng 2D.

⚡ Cải tiến — Dùng bảng 1D

💡 Ý tưởng cải tiến

Để tính dòng $i$, ta chỉ cần dòng $i-1$. Vậy ta không cần lưu cả bảng 2D — chỉ cần 1 mảng 1D $V[0..n]$ và cập nhật từ phải qua trái (để không ghi đè giá trị cũ trước khi dùng).

int Comb(int n, int k) {
    int V[n+1];
    V[0] = 1; V[1] = 1;        // dòng 0 và 1 của tam giác Pascal
    for (int i = 2; i <= n; i++) {
        int p1 = V[0];          // lưu V[j-1] cũ
        for (int j = 1; j < i; j++) {
            int p2 = V[j];      // lưu V[j] cũ trước khi ghi đè
            V[j] = p1 + p2;     // V[j] mới = V[j-1]_cũ + V[j]_cũ
            p1 = p2;            // dịch p1
        }
        V[i] = 1;               // biên phải
    }
    return V[k];
}

Bộ nhớ giảm: O(n²) → O(n). Thời gian vẫn O(n²).

2️⃣ Bài toán 2: Dãy con tăng dài nhất (LIS)

💡 Bài toán

Cho dãy $a_1, a_2, ..., a_n$. Tìm độ dài dãy con tăng dài nhất (Longest Increasing Subsequence — LIS). Dãy con phải giữ nguyên thứ tự các phần tử (không cần liền nhau).

Ví dụ: $a = \{1, 2, 3, 4, 9, 10, 5, 6, 7, 8\}$ → LIS = 8 (dãy $\{1, 2, 3, 4, 5, 6, 7, 8\}$).

📐 Công thức truy hồi

Định nghĩa: $F[i]$ = độ dài LIS kết thúc tại $a[i]$.

$$F[i] = \max(F[j]) + 1 \quad \text{với } j < i \text{ và } a[j] < a[i]$$

$$F[1] = 1 \quad \text{(chỉ gồm 1 phần tử)}$$

Kết quả: $\max(F[i])$ với $i = 1..n$.

Giải thích: Tại $a[i]$, ta nối $a[i]$ vào sau LIS dài nhất kết thúc tại $a[j]$ nào đó (với $a[j] < a[i]$). Lấy $F[j]$ lớn nhất rồi cộng 1.

🐍 Cài đặt Python

def lis(a):
    n = len(a)
    F = [1] * n          # F[i] = độ dài LIS kết thúc tại a[i]
    prev = [-1] * n      # để truy vết
    for i in range(1, n):
        for j in range(i):
            if a[j] < a[i] and F[j] + 1 > F[i]:
                F[i] = F[j] + 1
                prev[i] = j
    # Tìm vị trí có F lớn nhất
    max_idx = F.index(max(F))
    # Truy vết
    seq = []
    cur = max_idx
    while cur != -1:
        seq.append(a[cur])
        cur = prev[cur]
    seq.reverse()
    return max(F), seq

a = [1, 2, 3, 4, 9, 10, 5, 6, 7, 8]
print(lis(a))  # (8, [1, 2, 3, 4, 5, 6, 7, 8])

📊 Phân tích độ phức tạp

Thời gian: 2 vòng lặp for lồng nhau → O(n²).

Bộ nhớ: O(n) cho $F$ và $prev$.

Cải tiến: Dùng Binary Search để tìm $j$ nhanh → O(n log n). (Nâng cao, ít gặp trong đề thi cơ bản.)

3️⃣ Bài toán 3: Dãy con chung dài nhất (LCS)

💡 Bài toán

Cho 2 dãy $A$ và $B$. Tìm dãy con $C$ chung dài nhất của $A$ và $B$.

Ví dụ: $A = \{1, 2, 3, 4, 5, 6, 7\}$, $B = \{2, 4, 3, 5, 6, 8, 7, 1, 10\}$ → $C = \{2, 4, 5, 6, 7\}$, độ dài 5.

📐 Công thức truy hồi

Định nghĩa: $F[i, j]$ = độ dài LCS của $A[1..i]$ và $B[1..j]$.

$$F[i, j] = F[i-1, j-1] + 1 \quad \text{nếu } A[i] = B[j]$$

$$F[i, j] = \max(F[i-1, j], F[i, j-1]) \quad \text{nếu } A[i] \ne B[j]$$

$$F[0, j] = F[i, 0] = 0 \quad \text{(biên: 1 dãy rỗng → LCS = 0)}$$

Giải thích:

  • Nếu $A[i] = B[j]$: 2 phần tử này khớp → đưa vào LCS → $F[i,j] = F[i-1,j-1] + 1$.
  • Nếu $A[i] \ne B[j]$: phải bỏ 1 trong 2 → lấy max của $F[i-1,j]$ (bỏ $A[i]$) hoặc $F[i,j-1]$ (bỏ $B[j]$).

🐍 Cài đặt Python

def lcs(A, B):
    n, m = len(A), len(B)
    F = [[0]*(m+1) for _ in range(n+1)]
    for i in range(1, n+1):
        for j in range(1, m+1):
            if A[i-1] == B[j-1]:
                F[i][j] = F[i-1][j-1] + 1
            else:
                F[i][j] = max(F[i-1][j], F[i][j-1])
    # Truy vết
    seq = []
    i, j = n, m
    while i > 0 and j > 0:
        if A[i-1] == B[j-1]:
            seq.append(A[i-1])
            i -= 1; j -= 1
        elif F[i-1][j] >= F[i][j-1]:
            i -= 1
        else:
            j -= 1
    seq.reverse()
    return F[n][m], seq

A = [1, 2, 3, 4, 5, 6, 7]
B = [2, 4, 3, 5, 6, 8, 7, 1, 10]
print(lcs(A, B))  # (5, [2, 4, 5, 6, 7])

📊 Phân tích độ phức tạp

Thời gian: 2 vòng lặp lồng nhau kích thước $n$ và $m$ → O(n·m).

Bộ nhớ: O(n·m) cho bảng $F$.

Truy vết: đi từ $F[n][m]$ ngược về $F[0][0]$, mất O(n+m) bước.

4️⃣ Bài toán 4: Cái ba lô 0/1 (0/1 Knapsack)

💡 Bài toán

Ba lô trọng lượng $W$. Có $n$ đồ vật, mỗi vật $i$ có trọng lượng $g_i$ và giá trị $v_i$. Mỗi vật CHỈ CÓ 1 CÁI (chọn hoặc không chọn — đây là điểm khác biệt với Knapsack vô hạn ở bài Tham lam).

Mục tiêu: Chọn các vật sao cho tổng trọng lượng $\le W$ và tổng giá trị lớn nhất.

⚠️ Khác biệt với Knapsack vô hạn (Greedy)

• Knapsack vô hạn (Bài 3 — Greedy): mỗi vật có số lượng không giới hạn. Tham lam (chọn theo tỉ lệ $v_i/g_i$ giảm dần) cho kết quả gần tối ưu.

• 0/1 Knapsack (Bài này — DP): mỗi vật chỉ có 1 cái. Tham lam KHÔNG tối ưu → phải dùng DP.

📐 Công thức truy hồi

Định nghĩa: $F[k, V]$ = tổng giá trị lớn nhất khi xét $k$ đồ vật đầu, trọng lượng còn lại $V$.

$$F[k, V] = \max(F[k-1, V], \; F[k-1, V - g_k] + v_k)$$

$$F[0, V] = 0 \quad \text{(không có vật nào → GT = 0)}$$

$$F[k, 0] = 0 \quad \text{(TTL còn 0 → không chọn thêm)}$$

Giải thích: Tại vật $k$, có 2 lựa chọn:

  • Không chọn vật $k$: $F[k, V] = F[k-1, V]$ (giữ nguyên TTL, không thêm GT).
  • Chọn vật $k$ (nếu $g_k \le V$): $F[k, V] = F[k-1, V - g_k] + v_k$ (giảm TTL đi $g_k$, tăng GT thêm $v_k$).

Lấy max của 2 lựa chọn.

Giả mã (C) — theo giáo trình

int v[] = {0, 4, 5, 6, 3, 1};    // giá trị (v[0] bỏ qua)
int g[] = {0, 3, 4, 5, 2, 1};    // trọng lượng
int w = 10, n = 5;                // W = 10, 5 vật
int f[6][11];                     // f[k][V], k=0..5, V=0..10

// Khởi tạo dòng k=0
for (int i = 0; i <= w; i++) f[0][i] = 0;

// Tính bảng
for (int i = 1; i <= n; i++) {
    for (int j = 0; j <= w; j++) {
        f[i][j] = f[i-1][j];    // mặc định: không chọn vật i
        if (g[i] <= j && f[i-1][j] < f[i-1][j-g[i]] + v[i])
            f[i][j] = f[i-1][j-g[i]] + v[i];   // chọn vật i tốt hơn
    }
}
// Kết quả: f[n][w] = f[5][10]

📊 Phân tích độ phức tạp

Thời gian: 2 vòng lặp lồng nhau $n$ và $W$ → O(n·W).

Bộ nhớ: O(n·W) cho bảng 2D. Có thể tối ưu về O(W) bằng bảng 1D.

Lưu ý: Đây là độ phức tạp giả đa thức (pseudo-polynomial) — phụ thuộc vào $W$ (số nguyên). Nếu $W$ rất lớn thì không khả thi.

🔍 Truy vết

💡 Cách truy vết

Đi từ $F[n][W]$ ngược về $F[0][0]$:

  • Nếu $F[k][V] \ne F[k-1][V]$ → vật $k$ đã được chọn. Giảm $V$ đi $g_k$, giảm $k$ đi 1.
  • Nếu $F[k][V] = F[k-1][V]$ → vật $k$ không được chọn. Chỉ giảm $k$ đi 1.

5️⃣ Bài toán 5: Phân tích số

💡 Bài toán

Cho số $n < 100$. Đếm số cách phân tích $n$ thành tổng các số nguyên dương (thứ tự không quan trọng).

Ví dụ: $n = 5$ có 7 cách:

1. 5 = 1+1+1+1+1 2. 5 = 1+1+1+2 3. 5 = 1+1+3 4. 5 = 1+2+2 5. 5 = 1+4 6. 5 = 2+3 7. 5 = 5

📐 Công thức truy hồi

Định nghĩa: $F[m, v]$ = số cách phân tích số $v$ thành tổng các số nguyên dương $\le m$.

$$F[m, v] = F[m-1, v] \quad \text{nếu } m > v$$

$$F[m, v] = F[m-1, v] + F[m, v-m] \quad \text{nếu } m \le v$$

Giải thích:

  • Nếu $m > v$: không thể dùng số $m$ (vì $m$ đã lớn hơn $v$) → chỉ xét các số $\le m-1$: $F[m, v] = F[m-1, v]$.
  • Nếu $m \le v$: có 2 trường hợp — không dùng $m$ ($F[m-1, v]$) hoặc dùng ít nhất 1 lần số $m$ ($F[m, v-m]$). Cộng lại.

Biên: $F[1, v] = 1$ với mọi $v$ (chỉ có 1 cách: toàn số 1). $F[m, 0] = 1$ (1 cách: không dùng số nào).

🐍 Cài đặt Python

def count_partitions(n):
    # F[m][v], m=1..n, v=0..n
    F = [[0]*(n+1) for _ in range(n+1)]
    for m in range(1, n+1):
        F[m][0] = 1        # 1 cách: không dùng số nào
        for v in range(1, n+1):
            F[m][v] = F[m-1][v]
            if m <= v:
                F[m][v] += F[m][v-m]
    return F[n][n]

for n in [1, 2, 3, 4, 5]:
    print(f"n={n}: {count_partitions(n)} cách")
# n=1: 1, n=2: 2, n=3: 3, n=4: 5, n=5: 7

📊 Phân tích độ phức tạp

Thời gian: 2 vòng lặp kích thước $n$ → O(n²).

Bộ nhớ: O(n²) cho bảng 2D.

🏆 Cách trình bày bài thi DP

📋 Mẫu trình bày chuẩn cho bài DP

Đề bài mẫu: "Trình bày thuật toán DP giải bài toán cái ba lô 0/1. Chạy tay ví dụ."

Trình bày chuẩn:

1. NGUYÊN LÝ DP:
   DP giống chia để trị nhưng các bài toán con CHỒNG LẤP.
   → Ghi nhớ lời giải vào bảng, tính từ dưới lên (bottom-up).

2. ĐỊNH NGHĨA F:
   F[k, V] = tổng giá trị lớn nhất khi xét k đồ vật đầu,
             trọng lượng còn lại V.

3. CÔNG THỨC TRUY HỒI:
   F[k, V] = max(F[k-1, V],  F[k-1, V - gₖ] + vₖ)
   F[0, V] = 0   (không có vật → GT = 0)

4. GIẢ MÃ (C):
   for (i = 0; i <= w; i++) f[0][i] = 0;
   for (i = 1; i <= n; i++)
     for (j = 0; j <= w; j++) {
       f[i][j] = f[i-1][j];
       if (g[i] <= j && f[i-1][j] < f[i-1][j-g[i]] + v[i])
         f[i][j] = f[i-1][j-g[i]] + v[i];
     }

5. CHẠY TAY VÍ DỤ:
   W = 10, 5 vật: (g,v) = (3,4),(4,5),(5,6),(2,3),(1,1).
   Lập bảng f[6][11], điền từng ô...
   → f[5][10] = 12.

6. PHÂN TÍCH ĐỘ PHỨC TẠP:
   - Thời gian: 2 vòng lặp n × W → O(n·W).
   - Bộ nhớ: O(n·W) cho bảng 2D (có thể tối ưu O(W) bằng 1D).

7. TRUY VẾT:
   Từ f[n][W] ngược về f[0][0].
   Nếu f[k][W] ≠ f[k-1][W] → vật k được chọn, W -= gₖ.
   Với ví dụ: chọn vật 2, 3, 5 (TTL = 4+5+1 = 10, GT = 5+6+1 = 12).
💡 5 bí kíp để được điểm tối đa
  1. Luôn nêu nguyên lý DP trước — giám khảo cần biết bạn hiểu "tại sao dùng DP" (bài toán con chồng lặp → ghi nhớ).
  2. Định nghĩa F[i] (hoặc F[i,j]) RÕ RÀNG — đây là chìa khóa. Sai định nghĩa → sai cả bài.
  3. Liệt kê công thức truy hồi + giá trị biên — không chỉ công thức, phải có $F[0] = ?$, $F[1] = ?$.
  4. Chạy tay bảng trên 1 ví dụ nhỏ — chứng tỏ hiểu thuật toán. Điền từng ô, giải thích vì sao.
  5. Phân tích độ phức tạp + truy vết — đề thường yêu cầu "chỉ ra các vật được chọn" → phải biết truy vết.
⚠️ Lỗi mất điểm thường gặp
  • ❌ Định nghĩa F sai — vd LCS: $F[i,j]$ phải là độ dài LCS của $A[1..i]$ và $B[1..j]$, KHÔNG phải độ dài LCS của $A[0..i]$ và $B[0..j]$.
  • ❌ Quên giá trị biên — $F[0]$, $F[1]$, $F[0][j]$, $F[i][0]$... Phải khởi tạo trước khi chạy công thức truy hồi.
  • ❌ Nhầm 0/1 Knapsack với Knapsack vô hạn — 0/1: mỗi vật 1 cái (DP). Vô hạn: số lượng không giới hạn (Greedy).
  • ❌ Không truy vết — nếu đề yêu cầu "chỉ ra dãy LIS / LCS / các vật được chọn" mà không truy vết → mất 30-40% điểm.
  • ❌ Nhầm hướng đi trong bảng LCS — khi $A[i] = B[j]$ đi CHÉO (i-1, j-1); khi khác đi LÊN hoặc TRÁI tùy giá trị lớn hơn.
  • ❌ Sai công thức phân tích số — phải là $F[m, v] = F[m-1, v] + F[m, v-m]$ (dùng $F[m, v-m]$, không phải $F[m-1, v-m]$).
  • ❌ Quên cải tiến 1D — nếu đề yêu cầu tối ưu bộ nhớ, phải biết giảm 2D → 1D (cập nhật từ phải qua trái).
